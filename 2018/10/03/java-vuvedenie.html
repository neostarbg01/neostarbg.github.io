Въведение в Java

03 Oct 2018

Лекция 1

Кратка история на Java

Замислен 1991-1992 като платформено-независимo (cross-platform) разширение на C компилаторите

Официално обявен през Май 1995 от Sun Microsystems на конференцията Sun World’95

Характеристики

Java е опростен обектно ориентиран, интерпретиран, надежден, сигурен, платформено-независим, преносим, високо производителен, разпределен и динамичен език

Обектно ориентиран

  • Йерархия на класовете с единствен корен Object (всички класове наследяват класа Object)

  • Чисто обектно ориентиран

  • Класовете са организиарни в пакети (package)

Пакети
package com.google.program;

Конвенцията е сайта на компанията или програмиста, обърната (.com,net,org първо, след това името на сайта) и името на програмата

Ако нямаме сайт може да използваме me.<име>.program

Пример:

package me.neostarbg.calculator;

Пакетите действат като "папки" при организиране на кода. Свързани по между си класове са в един пакет

Различните пакети могат да включват клас със същото име, тогава различните класове се различават по пакетите

me.neostarbg.calculator.first.ClassName;
me.neostarbg.calculator.second.ClassName;

Строго типизиран

Съществуват типове променливи int, float, char както и тип на върнатата стойност на метод int method(){ return 3; }

Сигурен

Повишена надеждност

Концепцията за сигурност е вградена в езика - JVM (Java Virtual Machine):

Кодът се изпълнява във виртуална машина, не е възможно да повреди операционната система или хардуера

Платформено независим

“Write once, run everywhere”

-Мотото на Sun Microsystems

Изходният Java код се компилира до междинен bytecode Интерпретатора изпълнява този код

JRE (Java Runtime Environment) интерпретатор може да бъде изтеглен от сайта на Oracle за почти всяка операционна система

Основни типове приложения

Аплети

Applet - little application

Java проложения, вградени в HTML страница. Те се изпълняват от JVM вграден в браузъра

(вече не се използват никъде)

Конзолни проложения

Библиотеки

Ползват се от други Java приложения. Те се компилират и архивират в .jar файл

Jar файлове могат да се отварят със всеки архиватор (WinRAR, 7zip, etc.)

Структура на програмата

Входната точка е метод с име main()

Параметри от командния ред

При пускане на Java програма от командния ред могат да се задават параметри, които програмата да обработи и да има впредвид. Те се намират в стрингов масив като параметър на main()

Пример:

public static void main(String[] arguments)
{
    if(arguments.length > 0) //Ако броят аргументи са повече от 0
    {
        System.out.println("Hello, " + arguments[0]); //Hello <име>
    }
    else 
    {
        System.out.println("What is your name?");
    }
}

$java -jar myProgram.jar pesho
Hello, pesho

$java -jar myProgram.jar
What is your name?

Работа със стрингове

Стринговете са част от основната йерархия класове на езика - класа Strings

String name = "Pesho"; //String с главна буква за разлика от C#

Примерна Hello World програма - целия файл

package me.yourname.programname;

public class ProgramName
{
    public static void main(String args[])
    {
        System.out.println("Hello, world!");
    }
}
