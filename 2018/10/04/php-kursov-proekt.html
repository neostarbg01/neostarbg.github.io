Как създавам курсовия си проект по PHP

04 Oct 2018

Част 1 - 04/10/2018 11:09-20:55 (с много почивки >_>)

Ще използвам Laravel за да направя сайт за филми и игри под наем (не че някой още ги ползва ама защо не о3о). За име на сайта избрах CubeBuster (пародия на веригата за VHS касети под наем BlockBuster)

Използван софтуер

  1. Инсталирах Composer от https://getcomposer.org
  2. Вече имам git (не github десктоп приложението, трябва ни програмата Git Bash)
  3. Също така имам Visual Studio Code като текстов редактор
  4. Както и xampp пакет с Apache, Php и MySQL

Създаване на работната среда

  1. Стартирах Git Bash, отидох до cd /c/xampp/htdocs и направих директория mkdir laravel след което я отворих cd laravel

  2. Инсталирах Laravel и инициализирах проект composer create-project laravel/laravel cubebuster

  3. От C:\xampp\apache\conf\extra\ отворих httpd-vhosts.conf с нотепад, добавих следните данни на последния ред:

    <VirtualHost *:80>
    DocumentRoot "C:/xampp/htdocs/"
    ServerName localhost
</VirtualHost>
   
<VirtualHost *:80>
    DocumentRoot "C:/xampp/htdocs/laravel/cubebuster/public"
    ServerName cubebuster.net
</VirtualHost>

    След като стартирах Notepad++ (става и с нотепад) като администратор отидох до C:\Windows\system32\drivers\etc и отворих файла hosts. Добавих следния ред на нов ред: 127.0.0.1 cubebuster.net Сега вече мога, когато пусна сървъра, да отида на http://www.cubebuster.net от локалната машина и да се появи laravel сайта

  4. Отворих VS Code и с File->Open Folder отворих папката на проекта c:\xampp\htdocs\laravel\cubebuster\

  5. От File->Preferences->Settings в User Settings прозореца между двете къдрави скоби добавих "terminal.integrated.shell.windows": "C:\\Program Files\\Git\\bin\\bash.exe" заедно с кавичките. Ако има други user настройки трябва да се разделят със запетая

  6. Показах терминала от View -> Terminal

    Средата ми изглежда така:

Създаване на контролер и промяна на пътищата

  1. В терминала въведох $ php artisan make:controller HomeController

    mccre@DESKTOP-RJFGVST MINGW64 /c/xampp/htdocs/laravel/cubebuster
$ php artisan make:controller HomeController
Controller created successfully.
   
mccre@DESKTOP-RJFGVST MINGW64 /c/xampp/htdocs/laravel/cubebuster
$

    Controller created successfully. Новият контролер се намира във app/Http/Controllers наследява класа Controller който се намира в същата папка

    <?php
   
namespace App\Http\Controllers;
   
use Illuminate\Http\Request;
   
class HomeController extends Controller
{
    //
}

  2. Създадох функция index вътре, тя ще приема заявките към главната страница на сайта http://cubebuster.net/

    class HomeController extends Controller
{
	public function index()
    {
        return view('index');
    }
}

  3. Още не съм казал на laravel кой път ще води до тази функция обаче. В routes/web.php изтрих “welcome” Route-а и вместо него сложих

    Route::get('/', 'HomeController@index');

    Route означава че искаме да зададем функция, която да обработва даден път get е типа заявка, също може да е post, deletе, put, т.н

    Първия стринг ‘/’ е пътят. ‘/’ води до главната страница. ‘/about’ би водило до http://cubebuster.net/about. Втория стринг е името на контролера и името на функцията, които да обработят заявката, разделени с @

  4. Вече началната страница сочи до view-то ‘index’. Обаче то не съществува все още. За да го създам в resources/views създадох нов файл с име index.blade.php Той представлява нормален HTML файл, който може да обработва и PHP код както и blade templating синтаксис (както Razor от C# .NET)

    <html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="ie=edge">
    <link rel="stylesheet" href="">
    <title>Начало - </title>
</head>
<body>
    <h1>CubeBuster</h1>
    <h2>Най-добрите филми и игри - гледайте/играйте и върнете!</h2>
</body>
</html>

    3 забележки:

    • при извикването на стила вместо пътека до css файла (който е включен в laravel, включен е и Bootstrap) извикваме blade функцията asset($asset_path) между 4 къдрави скоби
    • Същото е и при <title> тага - този път финкцията config($option, $alt). Искаме да видим стойността на app.name, а ако няма такава, да изпише “CubeBuster”
      • Ако пусна сайта сега ще изпише “Начало - Laravel”. За да сменя името на проложението отворих файла ‘.env’ и промених APP_NAME на първия ред

Външен вид

  1. Създадох нов файл в /resources/sass с име _custom.scss
  2. В app.scss добавих ред на края на файла @import 'custom'; (името на файла без долна черта и .scss)
  3. В терминала на VS Code написах npm install и npm run watch за да компилира scss кода в чист css всеки път като правя промяна
  4. Използвах Bootstrap, Google Fonts, FontAwesome, particlesjs и постер на John Wick 2 за да създам външния вид на сайта

Следващия път ще се свържа с MySQL и ще направя админ потребител, може би ¯\(ツ)/¯
